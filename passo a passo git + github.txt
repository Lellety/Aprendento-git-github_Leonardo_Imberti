1°   
 → para adicionar um unico arquivo
 $ git add "nome do arquivo" 
 → para adcionar varios arquivos
 $ git add .   

2° 
→ para verificar os arquivo criado utilizar o cod.
$ git status

3° 
→ fazer Adicção de comentarios ao arquivo modificado
$ git commit -m " obs. Comentário desejado"

4° 
→ enviar o arquivo no github
$ git push


5° 
→ para verificar a versão ( atualização ) do seu cod. enviado para github ( histórico )
$ git reflog

6° 
→ como navegar entre as versões ( atualização ) dos cod enviado ao github ( obs. caso seja necessário em caso de erro nos cod editado)
$ git reset --hard ( obs. mais o cod. do Id da versão )

7°
→ para verificar as branch ( caminhos )
$ git branch

→ para adiconar uma branch ( caminhos )
$ git branch ( obs, mais o nome desejado )
ex: 
$ git branch staging

→ para selecionar o branch pra edição ou envio
$ git checkout staging

8°
→ para atualizar ( equalizar ) os branch ( caminhos ) apos as edição aprovadas.
tem que estar na branch que vai receber o arquivo atualizado e usar o comando de de onde vai puxar o arquivo
ex: estar na master e puxar da staging
$ git merge staging
